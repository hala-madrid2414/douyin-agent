## 🎯 阶段一：基础AI对话底座开发

### 1. 静态UI交互激活（Mock数据阶段）

- [ ] 将欢迎语修改为：「你好，我是你的专属旅行助手，随时为你规划行程、解答旅行问题～」
- [ ] 实现输入框交互：支持回车发送、点击发送按钮发送，空内容时禁用发送按钮
- [ ] 实现消息气泡渲染：区分「用户消息」与「AI消息」样式，用Mock数据展示示例对话
- [ ] 实现侧边栏「新对话」功能：点击后清空当前对话列表，开启新会话
- [ ] 实现本地缓存：用`localStorage`存储对话列表，刷新页面后自动恢复历史消息

### 2. 后端BFF与API对接（非流式阶段）

- [ ] 在Modern.js BFF层创建`/api/chat`接口，接收前端消息与对话历史
- [ ] 固化「专属旅行助手System Prompt」到接口逻辑
- [ ] 配置大模型API密钥，实现非流式调用：接收请求→携带Prompt→调用大模型→返回完整回答
- [ ] 编写基础异常处理：API超时/失败时返回友好提示（如「暂时无法为你规划行程，请稍后重试」）
- [ ] 联调前后端：将Mock数据替换为真实接口返回，验证AI按旅行助手角色回答问题、拒绝非旅行闲聊

### 3. SSE流式渲染与体验优化

- [ ] 改造`/api/chat`接口为SSE流式返回，逐片推送`content`字段，屏蔽`thinking`思考过程
- [ ] 前端实现SSE接收逻辑，用`EventSource`实现打字机效果（控制逐字显示速度）
- [ ] 添加加载状态提示：AI思考时展示「正在为你规划旅行…」
- [ ] 实现「一键跳过加载」功能：点击后终止打字机效果，直接展示完整回答
- [ ] 集成Markdown渲染：支持列表、加粗格式，优化行程/推荐类内容排版
- [ ] 优化UI适配：保证窗口大小变化时，消息气泡、输入框、侧边栏布局不乱

### 4. 阶段一验收与修复

- [ ] 核心流程测试：输入「国庆3天带娃去青岛，预算2000，求小众行程」→发送→AI流式输出分点行程
- [ ] 边界场景测试：验证非旅行问题会被礼貌拒绝，API失败时显示友好提示
- [ ] 缓存测试：刷新页面后对话历史完整保留，「新对话」功能正常
- [ ] 体验优化：修复排版错乱、加载状态不消失等细节问题

## 🎯 阶段二：Agent核心能力开发——Function Call工具调用

### 1. 工具封装与规范制定

- [ ] 封装Tavily联网搜索工具：定义入参、出参格式，编写工具执行逻辑
- [ ] 制定工具统一调用规范：明确工具标识、触发条件、入参校验、出参解析、成功/失败回调
- [ ] 单独测试工具：验证能精准检索旅行实时信息（如「北京香山当前枫叶状态」），返回结构化结果

### 2. LangGraph决策链路搭建

- [ ] 基于LangGraph.js构建Agent思考链路：接收问题→判断是否调用工具→选择工具→执行调用→解析结果→生成回答
- [ ] 制定工具触发规则：仅在需要实时/最新旅行信息时触发工具，基础旅行常识直接回答
- [ ] 融合原有对话链路：无需调用工具时走阶段一模型直答逻辑，需要时先调用工具再整合结果回答

### 3. 后端工具调度层开发

- [ ] 新增工具调度接口：实现「模型工具调用指令→解析→匹配工具→执行→返回结果」闭环
- [ ] 整合模型调用接口与工具调度接口，对外提供统一Agent能力接口
- [ ] 完善工具调用异常处理：工具调用超时/失败时自动降级为模型直答，并提示「暂时无法获取实时信息，为你提供基础建议」

### 4. 前端工具调用体验适配

- [ ] 新增工具调用状态提示：如「正在为你查询青岛实时天气…」「正在检索景区开放信息…」
- [ ] 实现工具结果融合渲染：将实时信息以自然文字形式呈现，关键信息高亮
- [ ] 新增「重新调用工具」功能：在AI消息气泡右下角添加按钮，支持手动触发工具重查
- [ ] 优化流式渲染：工具调用完成后，融合回答仍保持打字机效果，体验无断层

### 5. 阶段二验收与修复

- [ ] 工具调用测试：验证实时旅行问题能精准触发工具，非实时问题直接回答
- [ ] 结果融合测试：验证AI不直接展示原始工具数据，而是整合为专业旅行建议
- [ ] 异常降级测试：工具失败时自动降级，无页面卡死/无响应情况
- [ ] 体验优化：修复状态提示延迟、结果高亮错乱等问题

## 🎯 阶段三：Agent生态能力开发——MCP/Skills + RAG

### 1. MCP/Skills旅行场景化集成

- [ ] 按MCP协议封装旅行专属Skills：如亲子旅行规划、情侣旅行规划、户外徒步规划等
- [ ] 制定Skills触发规则：根据用户旅行场景（带娃/情侣/户外）自动匹配对应Skill
- [ ] 制定Skills优先级规则：多工具/Skills匹配时，优先触发专属Skill，再调用工具补充实时信息
- [ ] 统一Skills输出结构：如行程规划需包含「每日安排、核心亮点、交通方式、注意事项」

### 2. RAG检索增强能力开发

- [ ] 搭建旅行专属知识库：收集热门/小众目的地攻略、旅行避坑指南等，完成文档切分与向量化
- [ ] 封装「知识库检索」工具：制定触发规则（如涉及小众景点、避坑指南时触发）
- [ ] 实现RAG+Function Call融合：支持先检索本地知识库，再调用工具补充实时信息
- [ ] 实现知识来源标记：在AI回答中对知识库内容添加「【专属旅行攻略】」标记，前端差异化展示

### 3. 对话持久化与记忆升级

- [ ] 接入MongoDB数据库，实现对话数据云端存储：生成唯一对话ID，存储对话记录、用户旅行需求等
- [ ] 实现对话历史跨设备同步：整合本地缓存与云端存储
- [ ] 实现对话收藏功能：在单条对话气泡右侧添加收藏按钮，收藏的旅行攻略做特殊标记
- [ ] 升级上下文记忆能力：让Agent记住用户旅行偏好（预算、出行人数、偏好类型）

### 4. 前端体验适配

- [ ] 新增「收藏对话」「筛选对话」功能：支持按时间、收藏状态筛选历史对话
- [ ] 新增「清空云端对话」功能：添加二次确认提示，避免误操作
- [ ] 实现知识来源标记渲染：用不同字体颜色/图标区分知识库内容与通用回答
- [ ] 优化多能力调用状态提示：如「正在为你检索小众攻略+查询实时天气…」

### 5. 阶段三验收与修复

- [ ] Skills触发测试：验证亲子/情侣等场景能精准匹配对应Skill，输出结构统一
- [ ] RAG检索测试：验证小众旅行问题能触发知识库检索，结果准确且来源清晰
- [ ] 记忆与持久化测试：验证对话跨设备同步、收藏功能正常，Agent能记住用户偏好
- [ ] 多能力融合测试：验证复杂旅行需求（如「带老人去桂林，预算3000，求小众慢行程」）能灵活组合能力回答

## 🎯 阶段四：Agent体验升级开发——生成式UI

### 1. AI结构化输出约定

- [ ] 优化专属旅行助手Prompt，新增结构化输出要求：需要可视化时返回「文字回答+结构化数据+UI渲染指令」
- [ ] 制定UI渲染指令规范：
  - [ ] 行程规划→`render_travel_plan`+JSON格式行程数据
  - [ ] 美食/住宿推荐→`render_list`+JSON格式清单数据
  - [ ] 旅行贴士→`render_tips`+JSON格式贴士数据
- [ ] 验证AI输出准确性：确保结构化数据与文字回答一致，渲染指令无歧义

### 2. 前端旅行专属UI组件库开发

- [ ] 开发核心UI组件：多日行程时间轴组件、美食/住宿卡片组件、旅行贴士标签组件
- [ ] 统一组件设计风格：与原有对话界面保持一致，贴合旅行轻量调性
- [ ] 实现组件基础交互：行程组件可展开/收起，清单组件可查看详情，贴士组件可按重要程度筛选

### 3. 生成式UI渲染引擎开发

- [ ] 封装渲染解析逻辑：识别AI返回的渲染指令与结构化数据，自动匹配对应组件
- [ ] 实现「文字+UI」混合渲染：先打字机渲染文字回答，再加载UI组件
- [ ] 实现组件动态加载：按需加载对应组件，无渲染指令时不加载任何UI
- [ ] 优化组件布局：保证组件紧贴AI消息气泡，滑动时与文字同步展示

### 4. 体验优化与兜底

- [ ] 添加UI生成状态提示：如「正在为你生成可视化行程…」
- [ ] 实现「转为纯文字」功能：点击按钮隐藏UI组件，仅展示文字回答
- [ ] 实现UI生成失败兜底：解析错误/组件加载失败时，自动屏蔽UI，仅展示纯文字
- [ ] 优化加载体验：避免UI组件加载卡顿、排版错乱

### 5. 「UI交互→Agent响应」闭环实现

- [ ] 实现UI组件交互数据传递：用户操作组件后，将操作信息传给后端
- [ ] 后端Agent解析交互信息：结合历史对话与旅行偏好，生成对应回答
- [ ] 实现UI更新：如用户点击「增加美食节点」，Agent更新行程组件并补充美食推荐
- [ ] 验证闭环流畅性：操作UI后，Agent响应及时，UI更新无延迟

### 6. 阶段四验收与修复

- [ ] 生成式UI测试：验证行程规划/推荐类问题能自动渲染对应组件，「文字+UI」混合渲染正常
- [ ] 兜底机制测试：验证UI生成失败时自动降级为纯文字
- [ ] 交互闭环测试：验证用户操作UI后，Agent能生成贴合操作的回答并更新UI
- [ ] 体验优化：修复组件加载慢、交互延迟、排版错乱等问题
